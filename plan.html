The spine

Critical path & the rules that can't bend

The two things that matter most — the Discovery Engine feeding validated rules into the strategy docs, and benchmark agents proving the firm beats a nap — set the order. The capacity-gap universe screen is the single root both tracks grow from.

  1. The capacity-gap universe screen precedes everything. It defines both the engine's coverage and the Discovery Engine's search space. Build it once, share it across tracks.
  2. Benchmark agents must exist and be graded before any sector can graduate. They are the scorekeeper for the "beats the index" criterion.
  3. A sector graduates only when all four convergence criteria hold. No graduation flag is ever set by hand.
  4. Grading runs first each morning, before the day's scan — price-at-call captured at call time. No look-ahead, ever.
  5. Point-in-time universe membership (including delisted names) wherever historical data is touched — survivorship bias is worst in small/mid-caps.

Convergence

Per-sector graduation

A sector flips from pre-validation to validated only when all four hold. The setGraduationFlags job is the only writer of the flag. The firm converts one sector at a time.

01 · Discovery
Validated rules for the sector that survived out-of-sample testing.
02 · Driving prompts
Those rules are live in the sector's agent prompts — provenance and version match.
03 · Track record
Enough graded calls accumulated after the rules went live to show performance.
04 · Beats the nap
That performance beats the benchmark agents for the sector's small/mid universe.

Honest sequence

Recommended build order

The quickest visible win is already banked — the visual org tab shipped. So the highest-leverage moves now start at the root, not the UI.

  1. T1-P0 — capacity-gap universe + Polygon + env. The root dependency of both tracks. Build the screen well; everything keys off it.
  2. T1-P1 — the 500-agent engine. Resolve the free-key blocker, or run paid for now.
  3. T1-P3 (benchmarks) + T1-P2 (semis doc) — grading with benchmark agents and one hand-authored sector. Stand up the scorekeeper before anything claims edge.
  4. T2-P0 → T2-P2 — Discovery machinery to the spurious-signal rejection test. The cheapest way to know the process is honest.
  5. T1-P4 — finish structural pre-validation labeling on the org tab.
  6. T1-P5 — production deploy: cron + live chat + auto-publish, supervised.
  7. T2-P3 → Convergence — discovery writes validated rules; graduate the first sector when all four criteria pass.
  8. F — options expression on graduated capacity-gap names, once options data is bought.

This system is built to prove or disprove its own edge cheaply — not to look impressive. The Discovery Engine feeding validated rules into the strategy docs, and the benchmark agents proving the firm beats a nap, are the two additions that make the difference. Everything else is machinery in service of those two.
